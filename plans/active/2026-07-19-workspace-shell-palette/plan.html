Ghostwriter delivery plan

Workspace shell · activity rail + quick jump

Status: active
Branch: feat/canvas-structure-ux-3 (shared until shell branch split)
Owner: founder + agent

Intent

Make Ghostwriter’s overall chrome feel like a writing IDE (Cursor-inspired): a slim activity rail, open/close/resizable panels, ⌘P to jump to any manuscript component, and ⌘⇧P as the top command / MCP chat surface.

Out of scope for first slice: full Cursor multi-panel workspace persistence, renaming all modes to Build/Map/Write, Playwright (until founder verifies).

Acceptance criteria

  • Activity rail is glyph-first, denser, with mode + panel toggles (Draft/Canvas/Split/Reader, Structure, Chat, Jump).
  • ⌘P / Ctrl+P opens a top quick-jump palette listing books, chapters, scenes, and story records; choosing one selects it and opens Draft for scenes.
  • ⌘⇧P / Ctrl+⇧P opens a top command + MCP chat palette (not only a right dock).
  • Structure rail can be drag-resized when expanded; chat top panel height can be drag-resized.
  • Existing Map/Canvas board shortcuts remain; Esc closes palettes.

Tasks

  1. Pure jump/command index + filter helpers + Vitest.
  2. Palette UI component (jump vs command/chat).
  3. Wire global shortcuts + rail affordances in AuthenticatedProjectWorkspace.
  4. Reposition MCP chat into top command surface; keep rail toggle.
  5. Structure width + chat height resize handles.
  6. Update handoff / record log; founder browser verify.

Tests and verification

  • Unit: jump target indexing + filter ranking.
  • Manual: ⌘P jump to scene; ⌘⇧P send chat; resize structure; Esc dismiss.
  • pnpm --filter @ghostwriter/ui exec tsc --noEmit and focused vitest.

Documentation and ADR impact

Update PRODUCT keyboard map when accepted. No ADR unless chat transport changes. Living design: Mockups 3.0 remains Map-authoritative; this plan owns overall shell chrome and may later update Mockups 3.0 app-chrome notes.

Risks and decisions

Locked for slice: ⌘P = manuscript jump; ⌘⇧P = commands + MCP chat at top. Activity rail stays writing modes (D/C/S/R), not a VS Code clone of SCM/Extensions.

Open: whether to split a dedicated feat/workspace-shell-palette branch before PR.
