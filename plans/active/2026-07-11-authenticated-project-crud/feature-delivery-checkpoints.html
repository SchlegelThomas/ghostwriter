Autonomous delivery harness

Tree, workflows, reader, split, and handoffs

This supplement applies the project delivery skill to the founder-accepted tree + DAG direction, Book Reader, Split, and scene handoffs. Each checkpoint is independently observable, documented, reviewed for cumulative coherence, verified, and committed before the next begins.

Accepted product and architecture boundaries

  • Structure authority: Project → Book → Part → Chapter → Scene and Unassigned scenes define canonical manuscript order.
  • Relationship authority: Canvas objects/links and story-knowledge links form an interpretive DAG over canonical IDs. They never silently reorder Draft.
  • Prose authority: acknowledged scene working heads are canonical for the live manuscript; immutable checkpoints/variants preserve history.
  • Reader authority: Reader is a read-only projection over manuscript order and acknowledged scene documents. It takes no edit lease and creates no copy.
  • Concurrency: project metadata, scene working revision, and Canvas board retain separate optimistic preconditions.
  • Save truth: success toasts appear only after durable acknowledgement. Conflicts apply nothing and remain actionable.
  • Drill and workflows: drill depth, workflow lens, tree expansion, focus, and split ratio are view preferences—not canonical story state.
  • Current non-goals: collaborators/presence, comments/suggestions, real AI inference/image generation, full offline projects, import/export, purge, marketplace, and native freeform Canvas.

Checkpoint A — Manuscript tree, inspector, and acknowledgement toasts

Writer-visible outcome: open a project, navigate the persistent manuscript tree, rename/reorder/move a chapter or scene in context, and receive an action-specific acknowledged toast with safe Undo/Restore where available.

Domain and binding map

  • Reuse existing ProjectCommand types, owner policy, project version, archive/restore rules, and aggregate navigator. No new canonical records or migration.
  • Extract the existing manuscript-tree prototype into an authenticated tree with controlled selection, expansion, active/archived visibility, inline add/rename, keyboard reorder, and destination move.
  • Replace primary setup cards and horizontal scene chips with one tree plus a selection inspector. Keep complex destructive confirmation and refusal copy.
  • Add a central acknowledgement coordinator in the client. Project and Canvas success dispatch only after acknowledged responses; Draft keeps quiet save state plus one post-idle acknowledgement.
  • Bound metadata Undo to one client-held inverse command with a short expiry. Do not support Undo for create, empty-container removal, or multi-domain scene handoff. Canvas uses server snapshot Undo; archive uses Restore; Draft uses History.

Tests

  • Tree roles, levels, expand/collapse, inline rename, add, reorder, move, archive/restore, refusals, and keyboard parity.
  • No success toast before acknowledgement; safe Undo uses the latest project version; conflict invalidates Undo and stays visible.
  • Tree selection drives the same scene in Draft, Canvas, Split, and later Reader.
  • Existing core/storage/backend suites remain unchanged and green.

Checkpoint B — Layer drill and Canvas workflows

Writer-visible outcome: enter a chapter from the tree or Canvas, see its scenes and relationships become primary, move into a scene, return through breadcrumbs/Escape, and switch workflow lenses without mutating the manuscript.

Domain and design map

  • Drill scopes are project, chapter, and scene, referencing existing IDs only. The stack is client state.
  • Workflow lenses are Outline, Relationships, Continuity fixture, Plan → Draft, and Review. They filter/emphasize existing board/navigator projections.
  • Chapter bounds are derived from scene cards/regions. Camera transitions are 400–500 ms with instant reduced-motion fallback.
  • Relationships exposes inbound/outbound typed links and authority. Continuity uses deterministic fixture findings only. Plan → Draft opens Split at the same scene. Review exposes Canvas snapshots/Undo and provisional decisions.

Tests

  • Project → chapter → scene → parent; breadcrumb/Back/Escape; focus transfer; stale selected node; viewport retention.
  • Lens changes do not bump project or Canvas version until an explicit command executes.
  • Relationship direction, stale target, confirmed/provisional text, and chapter-scoped filtering.
  • Reduced motion and narrow ordered equivalent.

Checkpoint C — Book Reader

Writer-visible outcome: open Reader from the selected tree/Draft/Canvas scene, read the selected book in bound spreads on wide web or continuous chapters on narrow web, navigate by chapter and page, inspect scene-linked story context, then return to the exact prior scene and mode.

Domain map

  • Add a book-reader query projection over one BookId, canonical manuscript scene order, non-archived scenes, and acknowledged SceneDocumentHead values.
  • Reader is read-only, takes no scene lease, and stores no prose. Working draft is the initial pin. Immutable edition/checkpoint pinning remains a follow-on until complete project-level edition semantics exist.
  • Add batch scene-head reads in memory/Postgres and one bounded authenticated book-reader endpoint to avoid N scene requests.
  • Register book.reader.read with web/backend binding and an explicit authenticated-prose MCP exception.

Design map

  • Wide: two-page bound spread, chapter tabs, previous/next, page progress, running headers, typeset chapter opener, Links toggle, reduced-motion page turn.
  • Narrow: continuous scroll with chapter landmarks and next/previous scene controls.
  • Scene-level context rail initially surfaces Canvas/story links. Inline passage pins require a dedicated stable-block pin model and ADR before they appear; do not fake anchors by matching text.
  • Reader entry captures prior workspace mode, scene, tree expansion, drill scope, and Canvas selection; Exit restores them.

Tests

  • Multi-part/chapter/unassigned order, archived exclusion, empty document, authorization/non-disclosure, batch parity, payload bounds.
  • Wide spread/chapter navigation, keyboard arrows, progress, Links toggle, reduced motion, and Draft/Canvas round-trip.
  • Narrow scroll landmarks and exact scene return.
  • 50-scene reader projection/pagination smoke.

Checkpoint D — Split composition and complete scene handoffs

Writer-visible outcome: move from tree, Canvas, Reader, or chapter/scene lens into a focus-preserving Split; resize panes; write or inspect; and return without losing acknowledged prose, selection, viewport, or history context.

Handoff map

  • Storyboard-first: existing atomic project scene + genesis document + Canvas card transaction, now entered from the chapter lens/tree.
  • Write-first: place the selected Draft scene on Canvas and offer Open Split after acknowledgement.
  • Tree move: explicit manuscript move/reorder command; Canvas references stay stable and spine drift becomes a review prompt, never an automatic hint rewrite.
  • Reader: open at the same scene without a lease; return restores previous mode, selection, drill stack, and focus target.
  • Split: adjustable ratio persisted as an account/project preference; Draft remains mounted while visible so lease/caret survive Canvas selection changes that do not choose another scene.

Tests

  • Resize/reload ratio, medium collapse behavior, scene switch with dirty Draft, lease conflict, revision conflict, recovery, and focus/caret policy.
  • Atomic storyboard rollback; write-first placement; tree move plus stable Canvas card; reader round-trip.
  • No pre-ack toast and domain-specific conflict copy.
