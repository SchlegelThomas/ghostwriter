Accepted UX 2.1 direction

Build, Map, Write

Status: accepted 2026-07-18 — implementation authorized
Branch: feat/authenticated-project-crud
Parent plan: Authenticated writing workspace, revisions, and Story Canvas

This supplement responds to the 2026-07-17 live review of the real authenticated Cognition project. It continues the active epic rather than opening another branch. The founder accepted the complete hierarchy, Canvas, Draft, handoff, responsive, and accessibility direction on 2026-07-18. The interactive visual source is appended to Ghostwriter Mockups 2.0.

Intent

Turn Ghostwriter’s implemented hierarchy, Canvas, and scene-writing capabilities into three legible writer workflows: Build the manuscript without wrestling a tree, Map the story with visible Canvas tools, and Write on a calm manuscript page. The design keeps structure and metadata available without allowing either to dominate the prose.

The current screenshot proves the shell and canonical commands work, but it also exposes the main experience defect: a selected empty chapter leaves a mostly blank center while hierarchy actions hide in tiny row buttons and metadata fills the inspector. Canvas similarly has a large capability set below the fold, but no obvious tool model. Draft renders editor, recovery, lease, and history correctly, yet stacks them into one long technical panel instead of a writing desk.

Out of scope: collaboration presence, comments or tracked suggestions, AI generation, binary image upload, timeline simulation, native freeform Canvas, new canonical storage, automatic manuscript reorder from Canvas, and changing accepted version/lease/recovery semantics.

Proposed concepts

  • Story Trail: a persistent project → book → part → chapter → scene breadcrumb that doubles as a scope switcher. One coherent selection drives tree, center, and Context Dock; it preserves expanded tree state, Canvas scope, Draft caret, and Reader return context instead of leaving Draft pointed at an unrelated scene.
  • Quick Build: one context-aware Add action beside the tree and in empty states. It proposes the valid next records for the current selection, asks only for a title, applies a sensible placement, then keeps focus in the new inline title. Advanced placement remains available but never blocks the first action.
  • Scene Launchpad: when a chapter or Unassigned bucket has no selected scene, the center becomes useful: create a scene, move an existing unassigned scene here, storyboard on Canvas, or open the next scene. It never displays a dead “choose or create” card.
  • Canvas Workbench: a visible tool dock for Select, Hand, Scene, Note, Story record, Image reference, Region, and Connect; a utility bar for search, zoom, fit, arrange, history, and Spatial/Outline; and a contextual selection bar for common card/link actions. Project scope uses derived Chapter Aggregate cards; scene depth becomes a Scene Focus Stage with excerpt, links, and Draft/Split actions.
  • Thread Trace: choosing a link or relationship lens highlights one directed path, dims unrelated cards, labels direction and authority, and exposes an ordered text equivalent. It changes projection only.
  • Draft Desk: a centered Manuscript Page with a compact scene ribbon, contextual Whisper Toolbar, writer-language Save Truth strip, and a collapsible Context Dock. Draft History becomes an explicit drawer rather than permanent content below the editor.
  • Context Dock: one secondary panel with tabs appropriate to the current work: Brief, Story, Canvas, and History in Draft; Details, Links, and History in Canvas. The dock opens for a task and can be dismissed to give the work surface room.
  • Focus Halo: a reversible writing posture that hides the tree and Context Dock while preserving the scene ribbon, save truth, and an obvious exit. It never releases the scene lease or remounts the editor.

Acceptance criteria

Build — manuscript hierarchy

  • The tree remains the canonical order navigator, but row chrome is quiet until hover, focus, selection, or touch action. Selection is visually distinct from edit mode and drag state.
  • Selecting a project, book, part, or chapter updates the center and Context Dock together. Draft never silently auto-opens a scene outside the selected branch; opening or creating a scene is an explicit writer action.
  • A single ＋ Add control opens a context menu containing only valid actions for the selection: Book, Part, Chapter, Scene, or Story record. Keyboard shortcut ⌘/Ctrl + Enter opens the same menu.
  • Quick Build creates one record with a title and contextual parent. Enter commits; Escape cancels; acknowledgement returns focus to the new row. “More options” opens placement/status fields without making them mandatory.
  • Chapter and Unassigned empty states provide prominent Create scene, Move existing scene, and Storyboard on Canvas actions. Selecting a populated chapter shows a scene launch list rather than blank space.
  • Drag-and-drop communicates destination and before/after position. A searchable Move menu and Alt/Option + Arrow remain equivalent. Invalid destinations explain why nothing changed.
  • Rename is inline. Archive/remove is grouped under an overflow menu with explicit confirmation and refusal copy. Archived records appear through one filter, never mixed silently with active work.

Map — Story Canvas

  • Canvas opens with its tool dock and utility bar visible above the work surface; creation controls are not buried in stacked forms.
  • Select (V) and Hand (H/Space) are distinct. Scene (S), Note (N), Story record (K), Image reference (I), Region (R), and Connect (L) have labels, tooltips, pressed state, and screen-reader names.
  • Choosing a creation tool updates the cursor and provides one-line instruction. Clicking the board creates or opens the smallest required popover; Escape cancels without mutation.
  • Scene creation asks for title and explicit manuscript placement, then atomically creates the canonical scene, genesis Draft, and Canvas card. Coordinate, size, z-index, and story-order hint are automatic defaults; advanced geometry stays in Details.
  • Connect mode chooses source, target, link kind, authority, and optional label in a short staged flow. Direction is visible on the board and written in the ordered equivalent.
  • Selected objects expose a compact floating bar: Open Draft, Link, Duplicate note metadata where safe, Bring forward/back, Archive, and More. Detailed metadata stays in Context Dock.
  • Search/Jump, Fit selection, Fit board, zoom, minimap, alignment/arrange, snapshot history, and Spatial/Outline are visible utilities. Unsupported multi-select/grouping is not implied until implemented.
  • Workflow lenses remain Outline, Relationships, Continuity fixtures, Plan → Draft, and Review. The tool dock controls actions; lenses control projection and emphasis. Neither becomes a second canonical store.
  • Project lens renders derived Chapter Aggregate cards with scene/link counts and explicit dive affordances; these are projections over the tree/board, not new canonical object kinds.
  • Scene lens renders a Scene Focus Stage with acknowledged prose excerpt, inbound/outbound context, and Open Draft/Open Split/History actions using the same scene ID.
  • The reading-order spine remains derived from Draft. Drift can be reviewed, but applying a manuscript reorder requires an explicit structure command and confirmation.

Write — Draft Desk

  • Selecting a scene places focus in one centered manuscript page with comfortable line length. The screen title is the scene title, not “Shape the manuscript.”
  • The scene ribbon shows Story Trail, scene title, status/POV summary, previous/next scene, word count, and quiet Saving/Saved/Read-only state. Lease detail moves behind status unless attention is required.
  • The formatting toolbar is compact, sticky, keyboard-operable, and contextual. It uses writer-facing controls for paragraph, headings, emphasis, quotation, scene break, undo, and redo without exposing editor internals.
  • Context Dock defaults to Brief and can switch among Story, Canvas, and History. Metadata commits are acknowledged, but prose remains wider and visually primary.
  • Draft History is closed by default and opens as a drawer with Timeline, Named variants, and Compare. Opening it preserves editor mount, caret, scroll, lease, and unsaved recovery. Checkpoint, variant, compare, and restore retain current server-authoritative semantics.
  • Focus Halo hides tree and dock without remounting Draft. Escape or the visible Exit focus action restores the exact prior shell state.
  • Draft → Canvas, Canvas → Split, Draft → Reader, and Reader return preserve canonical scene ID, caret/selection where applicable, tree expansion, Canvas drill stack/selection, and focus target. Selecting a non-scene Canvas object in Split never remounts Draft or releases its lease.

Responsive and accessible

  • Wide ≥1240 px: rail + resizable tree + work surface + dismissible Context Dock. Draft page is centered independently of panel widths.
  • Medium 760–1239 px: center remains primary; tree and Context Dock are mutually exclusive drawers, never two hidden regions the writer must toggle blindly. Split intentionally becomes one primary pane with a quick switch.
  • Narrow <760 px: Project, Draft, Canvas outline, and Reader are top-level modes. Quick Build uses a bottom sheet; Canvas is ordered, not freeform; Draft is a single continuous page with a bottom formatting tray.
  • Every pointer interaction has a keyboard and screen-reader path. Focus returns to the initiating control after sheets/drawers close. Hierarchy depth, relationship direction, authority, save state, and invalid destinations are expressed in text.
  • Reduced motion removes camera/zoom transitions and panel slides while preserving scope/focus changes.

Required states

  • Structure: empty project, empty part, empty chapter, populated chapter, Unassigned, filtered search, inline create/rename, dragging, invalid drop, busy, conflict, archived filter, safe-removal refusal.
  • Canvas: first-open guidance, empty scope, populated board, active tool, selected card/link/region, connect source/target, provisional, dismissed, archived, stale canonical reference, viewport culling, large board, save failure, board conflict, history restore confirmation.
  • Draft: no scene, loading, editable lease, read-only archived, lease conflict/expiry, saving, acknowledged, retryable failure, revision conflict, recovery offer, empty history, comparison, restore confirmation, Focus Halo.
  • Handoffs: pending Draft flush, successful mode switch, failed flush with recovery retained, Reader return, medium-pane swap, narrow back navigation.

Implementation sequence after founder acceptance

  1. Checkpoint H1 — Story Trail + Quick Build: coherent structural/scene selection, context-aware create menu, inline title flow, useful chapter/Unassigned launch states, clear row affordances, destination menu, and medium tree/Context drawers while retaining canonical commands.
  2. Checkpoint H2 — Draft Desk: Manuscript Page composition, scene ribbon, Whisper Toolbar, Save Truth strip, Context Dock, Draft History drawer, and Focus Halo without remounting Tiptap or releasing the lease.
  3. Checkpoint H3 — Canvas Workbench: visible tool/utility bars, staged creation/link flows, contextual selection bar, fit/search/minimap, Chapter Aggregates, Scene Focus Stage, and progressive-disclosure inspector over existing Canvas commands.
  4. Checkpoint H4 — Handoffs + responsive posture: exact state/focus preservation across Draft/Canvas/Split/Reader, medium drawers, narrow bottom sheets/ordered Canvas, reduced motion, and keyboard parity.
  5. Checkpoint H5 — Coherence and polish: visual rhythm, loading/empty/error/conflict states, acknowledgement copy, full browser regression, docs, and handoff.

Tests and verification

  • New or materially rewritten unit, contract, integration, and post-acceptance Playwright tests route through routine-tests (Composer 2.5 fast). Use hard-tests (Grok 4.5) only for a recorded hard case or after two evidence-backed Composer attempts.
  • Quick Build: valid action matrix per selection, inline commit/cancel, acknowledged focus handoff, invalid destination, conflict/refusal, and exact canonical order.
  • Draft Desk: editor mount identity, caret/scroll preservation, Focus Halo, History drawer, save/lease/recovery truth, toolbar keyboard operation, and large-scene behavior.
  • Canvas Workbench: keyboard tools, staged create/link cancel/success, coordinate defaults, no implicit manuscript reorder, ordered parity, viewport utilities, selection visibility, and snapshot recovery.
  • Responsive: wide/medium/narrow visual and interaction journeys, no horizontal overflow, drawer/sheet focus containment and return, reduced motion, and screen-reader labels.
  • During H1–H5, run focused automated checks, affected package typechecks, pnpm verify, direct wide/medium/narrow browser walkthroughs, web export, and changed-document link/structure checks. After the founder verifies the complete outcome, audit and update the smallest high-value Playwright acceptance set, run focused green first, then one full pnpm test:e2e.

Documentation and ADR impact

This is a UX composition and binding iteration over accepted domain boundaries. Update the parent plan, record log, living design, product experience notes, README/handoff, and browser acceptance as checkpoints land. No ADR is required for the proposed shell itself.

An ADR or explicit founder decision is required before adding canonical grouping/multi-select semantics, persisted UI layout beyond existing personal preferences, automatic Canvas-to-manuscript reorder, a new timeline domain, or inline passage pins. This design does not assume those decisions.

Risks and decisions

  • Progressive disclosure: hiding advanced controls must not remove discoverability. Primary tools remain visible; More/Details houses low-frequency geometry, media metadata, history, and destructive actions.
  • Editor stability: drawers, Focus Halo, Split changes, Context Dock tabs, Reader round-trips, and non-scene Canvas selection must not remount Tiptap, lose caret, or churn the direct-edit lease.
  • Canvas scope: multi-select, grouping, snapping, and arrangement are useful but require precise command/history semantics. The visual design marks unsupported items as later instead of faking them.
  • Hierarchy speed: context-aware defaults reduce steps, but every resulting parent and manuscript position must be visible before acknowledgement.
  • Review gate: these concepts remain proposed until the founder accepts the visual pass. Product implementation does not begin merely because the design file renders.
