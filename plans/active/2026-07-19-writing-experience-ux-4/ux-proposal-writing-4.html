Writing Experience 4.0

Status: proposed — awaiting founder design acceptance
Parent plan: Writing Experience UX 4.0
Interactive source: Ghostwriter Mockups 4.0
Lineage: agentic writing blueprint · agent contracts · shipped Write/Canvas kernel · Mockups 2.0 / 3.0

This epic puts writing back at the center: a focused manuscript studio, craft companions (sketches, character sheets, backdrops), bounded writing agents, and input modes for keyboard, dictation, and stylus — integrated with Canvas and Split views.

Chrome contract: Mockups 4.0 visuals are synced to the live app (2026-07-19) — dense 44/36 toolbars, 32px rail, Map board spacing — not older mockup shells. Concepts unchanged; re-review the interactive studio first.

Intent

Writers should feel they are crafting a scene — inventing intent, casting people, setting place, then writing — with optional AI partners that propose, never silently author. Canvas remains the spatial sibling; Split lets craft or Map sit beside prose without remounting the scene lease.

Feature descriptions

  1. F1 · Writing Studio — Draft Desk gains Focus, Assist, clear lease/save chrome, and a composition switcher (Page / Split · Map / Sheet / Backdrop). Formatting stays minimal; craft and agents get the attention.
  2. F2 · Scene Sketch — Purpose, conflict, turn, beats, optional ink pad. Planned craft authority. Can pin to Canvas as a linked card.
  3. F3 · Character Sheet — Deepens story-knowledge characters (desire, pressure, voice, evidence). Split · Sheet shows active cast beside Draft.
  4. F4 · Backdrop — Extends scene ambience into a place/atmosphere companion for Split and Canvas pins. Never auto-dumps setting into prose.
  5. F5 · Writing Agents — Scene Partner, Character Coach, Worldkeeper, Sketch Partner. Assignment → proposal → compare → apply/dismiss. MCP parity required.
  6. F6 · Input Modalities — Keyboard default; dictation inserts into Tiptap; stylus writes on sketch/ink craft layer; free-text capture can land in notes/sketch before promotion.
  7. F7 · Split compositions — Named layouts sharing one scene lease. Narrow web stacks; no fake dual-pane Map under ~900px.
  8. F8 · First-book path — Optional guided spark → sketch → sheet → first scene. “Start writing” always available. Phase B+.

Editor recommendation

Keep Tiptap / ProseMirror as the canonical manuscript engine. It already carries schema-v1 JSON, stable block IDs, checkpoints, variants, and compare. Replacing it would reset the trust surface for negligible modality gains.

  • Dictation: speech → editor insert commands (browser SpeechRecognition first; managed STT later if quality demands).
  • Stylus: craft ink on Scene Sketch (recommended). Optional freehand-in-document is a later spike with schema risk.
  • Agent edits: never drive the live editor directly; emit revision-bound proposals.
  • Reject for this epic: Lexical/Slate/Quill migration, code editors, native-only ink apps as the primary web substrate.

Canvas & Split integration

  • Mockups 3.0 continues to own Map tools, structure collapse, and layer camera.
  • Write 4.0 owns Draft chrome, Assist, craft panes, and modality controls.
  • Opening Split from a Canvas scene card mounts the same scene ID; board gestures stay on Canvas version domain.
  • Sketch / character / backdrop objects may appear as Canvas cards via canonical IDs — not duplicate truth.
  • Draft remains authority for reading order.

Agent beachhead

Ship Scene Partner first (continuation / beat variants) once craft context and proposal UI exist. Character Coach and Worldkeeper follow sheet/backdrop persistence. Sketch Partner converts messy notes/ink summaries into sketch fields only.

All roles inherit the permanent authority rule from agent contracts.

Implementation sequence after acceptance

  1. Phase A — Writing Studio chrome (Focus, Assist shell, composition switcher)
  2. Phase B — Craft artifacts (Sketch, Sheet depth, Backdrop split + Canvas pins)
  3. Phase C — Dictation v1
  4. Phase D — Scene Partner proposals
  5. Phase E — Coach + Worldkeeper
  6. Phase F — Stylus craft + optional convert-to-text proposals

Each phase becomes its own accepted delivery plan with tests, docs/ADR updates, and founder verification before Playwright expansion.
