Ghostwriter delivery plan

Writer workspace depth

Status: active — autonomous delivery
Branch: feat/authenticated-project-crud

Intent

Respond to the 2026-07-12 live review of the Consciousness project. Writers need named scene folders with objectives/cast notes, drag-and-drop structure and Canvas placement, deeper story knowledge and typed relationships, scene ambience (backdrop/music/images), a book-like Reader with ElevenLabs voice selection, and an in-app MCP chat surface for capabilities. OpenAI keys arrive later; the chat shell must work with tool invocation first.

Out of scope for this plan: binary R2 upload (URL/metadata first), real OpenAI inference until the founder provides a key, collaboration presence, marketplace, native freeform Canvas redesign beyond layer coords and links.

Accepted product decisions

  • Scene folders = Chapters (and Parts for higher structure). Chapters gain title + summary/objectives/cast notes. Unassigned remains the system bucket for unplaced scenes and surfaces a clear “Create chapter folder” path rather than becoming a renameable record.
  • Media v1 = URL + metadata references on scenes (backdrop, music, image refs). R2 binary upload is a follow-on ADR; Canvas image-reference stays metadata-compatible.
  • Story knowledge depth adds a notes/body field, optional aliases, and knowledge↔knowledge links with kinds (cast, theme, development-cycle, breadcrumb). Scene links and Canvas object links remain.
  • Canvas layer coordinates persist as scope-keyed placements (project/chapter/scene) over the same object IDs—interpretive layouts, not a second manuscript order.
  • Reader TTS follows the wrapper ElevenLabs pattern: server-only synthesis, voice packs, client playback, graceful silence when unconfigured.
  • MCP chat is an authenticated in-app panel that lists Ghostwriter capabilities and can invoke registered tools; LLM routing waits on the founder’s OpenAI key.

Acceptance criteria

  • Chapter (scene folder) inspector edits title and objectives/description; toasts acknowledge; tree shows the summary snip.
  • Tree supports pointer drag-and-drop reorder/move of scenes (and keyboard Alt+Arrow parity remains); Canvas supports drag placement and creating typed links between story-knowledge cards and scenes.
  • Each Canvas drill layer retains distinct x/y placements for the same objects without changing manuscript order.
  • Scene inspector edits backdrop URL/caption, playable music URL/label, and related image URL refs; Reader and Draft can surface them.
  • Story knowledge records support notes body, aliases, and knowledge↔knowledge links with explicit kinds; Canvas Relationships lens highlights them.
  • Reader looks like a bound book (serif spreads, page chrome, chapter openers) and offers ElevenLabs voice pack selection with play/stop for the current spread/scene.
  • A docked MCP chat panel lists capabilities, runs read tools against the open project, and reserves a clear slot for later OpenAI tool-use.

Checkpoints

  1. A — Scene folders: chapter summary/objectives domain + inspector + navigator projection.
  2. B — Drag and drop: tree DnD move/reorder + Canvas object drag and link creation UX.
  3. C — Layer coords + knowledge graph: scope-keyed placements; knowledge notes/aliases/links; Relationships lens.
  4. D — Scene ambience: backdrop/music/image URL fields; inspector; Draft/Reader surface.
  5. E — Book Reader + ElevenLabs: visual book chrome; voice packs; server TTS route.
  6. F — MCP chat panel: docked UI; capability list; tool invoke against open project.
  7. G — Coherence: verify, docs, ADRs, checkpoint commits.

Documentation and ADR impact

  • ADR 0008 — chapter folder metadata, scene URL media refs, knowledge depth and K↔K links.
  • ADR 0009 — Canvas scope placements, ElevenLabs Reader TTS, in-app MCP chat shell.
  • Update Product, Architecture, API, Operations, living design notes, and handoff as each checkpoint lands.

Living design source: Ghostwriter Mockups 2.0 remains authoritative for Canvas/Draft/reader posture; this plan extends it with folder depth, ambience, TTS, and chat without replacing founder-authored screens wholesale.

Risks and decisions

  • Unassigned rename rejected: synthetic bucket stays; chapters are the named folders.
  • URL media can break or go stale; UI labels them as writer-owned references, not hosted assets.
  • ElevenLabs key must stay server-side (Fly secret / local env); never in the client bundle.
  • OpenAI chat completion is gated until the founder supplies a key; tool-only chat ships first.
