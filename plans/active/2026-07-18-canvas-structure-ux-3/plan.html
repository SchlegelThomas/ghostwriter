Ghostwriter delivery plan

Canvas & Structure UX 3.0

Status: proposed — design review
Branch: feat/canvas-structure-ux-3
Owner: founder + agent
Living design: Ghostwriter Mockups 3.0
UX contract: Canvas 3.0 proposal

Intent

Make Map mode feel like a real spatial board: collapsible manuscript structure for Canvas real estate, free pointer drag and pan, toolbar + right-click creation, drag-to-link relationships, and layer dive with zoom—while keeping keyboard shortcuts and explicit scene manuscript placement.

Out of scope: collaboration presence, AI generation, binary image upload, Canvas-driven automatic manuscript reorder, provisional scenes without book/chapter-or-Unassigned, Draft Desk redesign, and product implementation before founder design acceptance.

Acceptance criteria

Design gate (this slice)

  • Mockups 3.0 browsable with Map default collapsed structure + hidden Details, icon toolbar/utilities with shortcut tooltips, full keyboard map, context menu, link-drag preview, free-drag prototype, and layer zoom.
  • Navigation / real-estate section documents chrome budget and structure states (collapsed / peek / expanded).
  • Full journeys documented for create / link / move / edit / archive-delete / combo / layer enter-exit.
  • Explicit scene placement locked; freeform applies after creation.
  • Delivery plan + UX contract record invariants and post-acceptance tasks.
  • Handoff points at Mockups 3.0 for Map/structure work; Mockups 2.0 remains shipped 2.1 reference.

Implementation gate (after founder acceptance)

  • Wide Map defaults to ~36px collapsed rails + Details dismissed; intuitive »| / |« expand controls; [/] toggle; Story Trail in a compact top bar.
  • Icon-only edge toolbar (~36px) and utility strip with tooltip = Name · shortcut for every control; thin instruction strip.
  • Chrome and layer transitions use quiet ease-out motion; reduced-motion falls back to instant meaning-preserving changes; shell motion never remounts Draft.
  • Full keyboard map shipped (tools, chrome, viewport, nudge) with parity to tooltips and context menus.
  • Right-click / long-press menus on empty board, card, region, and link mirror toolbar actions; destructive paths confirm.
  • Select-drag freely moves cards/regions on wide web; Hand/Space pans; keyboard nudge remains; Fit utilities recenter.
  • Relationship create via handle-drag and Connect tool; Escape cancels; direction/authority remain explicit.
  • Double-click / Enter layer zooms into chapter/region/scene scope; Esc/Back/breadcrumbs exit; reduced motion skips zoom.
  • Scene create still requires title + book · chapter/Unassigned before acknowledgement.
  • Manuscript order stays canonical; Canvas version domain stays separate; no silent Draft remount from non-scene selection.
  • Narrow retains ordered Canvas (no freeform claim) with action sheets for create/connect.

Risks and decisions

  • Locked: explicit scene manuscript placement; design + plan before product code.
  • RN web drag flake: known from prior e2e; H2 must prove pointer drag in a real browser, not only nudge.
  • Menu vs dock parity: every context action needs toolbar/keyboard equivalent.
  • Zoom vs reduced motion: motion is progressive enhancement; scope change must work without it.
  • Preview cookies / domain: unrelated; production remains ghost-writer.studio.
