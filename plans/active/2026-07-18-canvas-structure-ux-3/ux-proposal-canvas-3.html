Proposed UX 3.0 direction

Canvas & Structure 3.0

Status: proposed — awaiting founder design acceptance
Branch: feat/canvas-structure-ux-3
Parent plan: Canvas & Structure UX 3.0
Interactive source: Ghostwriter Mockups 3.0

This supplement continues after shipped Build · Map · Write UX 2.1 (PR #6). It does not reopen Draft Desk. It makes Map mode spatial and discoverable: navigation chrome shrinks so the board owns the viewport, tools are icon-driven with shortcut tooltips, structure collapses by default, and layers keep dive/zoom meaning.

Intent

Writers should feel they are arranging a board—not filling stacked forms. Map navigation defaults to a thin project rail, collapsed structure glyphs, and Details hidden. Creation happens from a compact icon toolbar (tooltips show name + shortcut), context menus, or the full keyboard map. Relationships can be pulled between cards. Double-click zooms into a layer. Draft remains the authority for reading order; scene creation still names an explicit manuscript home before the scene exists.

Concepts

  • Map real-estate navigation: ~36px icon project rail; structure collapsed (~36px) by default with a clear »| expand control (and matching |« when open); [ toggles; Story Trail in a ~40px top bar.
  • Structure rail: collapsed / peek / expanded with eased width. Quick Build only when expanded (or ⌘/Ctrl+Enter while open).
  • Icon edge toolbar (~36px): Select, Hand, Scene, Note, Story, Image, Region, Connect as icons; hover/focus tooltip = Name · shortcut; thin instruction strip under the top bar.
  • Icon utility strip (~36px): lenses, Jump, Fit, zoom, History, view mode, Details toggle (]) — all with tooltips.
  • Motion & feel: soft ease-out (~180–450ms) for structure, Details, layer zoom, menus, and card settle; one primary motion at a time; honor prefers-reduced-motion; never remount Draft for chrome animation.
  • Context menus: right-click / long-press / ⇧F10; shortcuts listed on rows; parity with toolbar.
  • Free drag: Select-drag moves objects; Hand/Space pans; keyboard nudge and Fit remain.
  • Link drag: out-handle rubber-band to a target, plus Connect tool staged flow.
  • Layer zoom: double-click / Enter layer camera; Esc/Back out; reduced motion snaps.
  • Details dismissed by default while mapping; geometry and rare metadata live there when shown.

Keyboard map (required in mockup + product)

  • Tools: V H S N K I R L, hold Space to pan, Esc → Select / cancel.
  • Chrome: [ structure, ] Details, / Jump.
  • Viewport: +/− zoom, ⇧1 Fit selection, ⇧2 Fit board, arrows nudge.
  • Every icon control exposes the same shortcut string in its tooltip and accessible name.

Locked product rules

  • Scene create requires title + book · chapter-or-Unassigned before acknowledgement.
  • No provisional Canvas-only scenes in this slice.
  • No silent manuscript reorder from Canvas positions.
  • Canvas and Draft version domains stay separate.
  • Keyboard shortcuts from UX 2.1 remain first-class.

Journeys the mockup must make obvious

  1. Create note / region / story card / scene (with placement sheet).
  2. Free move + pan + nudge + Fit.
  3. Connect via handle drag and via Connect tool.
  4. Edit label; archive card; delete link with confirm.
  5. Collapse structure → create → link → rearrange → delete link → enter chapter → exit.
