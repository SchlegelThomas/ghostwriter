Sequencing rules

  1. One vertical outcome at a time. Every plan includes domain logic, UI, MCP binding or documented exception, tests, docs, and an observable writer result.
  2. Trust before intelligence. Persistence, recovery, revisions, proposals, and export exist before a real model can suggest a canonical change.
  3. Deterministic before probabilistic. Stable IDs, schemas, search, policy, evidence, and fixtures precede model evaluation.
  4. Web is the product, wrappers are conveniences. Prove the full responsive collaborative flow in the browser; add desktop or native capabilities only where they materially improve it.
  5. No phase advances on code completion alone. Reliability, usability, quality, and ownership gates matter as much as pnpm verify.
  6. Backlog is not commitment. Candidate plans are intentionally not active folders or branches until accepted.

1.5 · feat/import-export-portability

1.6 · feat/story-canvas-foundation

One spatial story model beside the page

Replace Plan with a zoomable Story Canvas for scenes, notes, characters, imagery, threads, regions, outline, and timeline views. Add write-first/storyboard-first entry, stable pins, split view, the reading-order spine, and provisional ghost objects.

Implemented locally under ADR 0007: Canvas and Draft share scene/story IDs, storyboard handoff is atomic, the spine exposes drift without reordering Draft, and narrow web has an ordered accessible alternative. Timeline and AI derivation remain later.

Phase gate: a collaborative web product with AI turned off

  • An author can create or import a multi-book project, write for a long session, invite an editor, resolve tracked suggestions, compare a variant, create an edition, and export.
  • Crash, reconnect, authorization, conflict, and migration fixtures lose zero acknowledged edits.
  • Representative imports preserve at least 95% of supported headings, paragraphs, emphasis, and scene boundaries.
  • All proposal safety invariants pass with fixture data.
  • Story Canvas and Draft stay linked without duplicated scene truth; large-book and three-book performance budgets pass.
  • Responsive web critical paths are usable with keyboard and screen reader; wrappers remain optional.

Phase 2 — Project-aware intelligence

2.1 · feat/story-model

Story bible, facts, timeline, and threads

Add typed entities, relationships, planned/confirmed/inferred/disputed facts, fictional timeline events, narrative threads, evidence links, mentions, and history.

Exit: story views and MCP resources read the same core records; authority states cannot be confused.
2.2 · feat/project-search-context

Local search and context receipts

Add revision-aware lexical search, deterministic workflow context assembly, token budgets, source reasons, derived summaries, invalidation, and a writer-visible context receipt.

Exit: same inputs produce the same context; deleting derived indexes loses no project data.
2.3 · feat/agent-safety-kernel

Capabilities, consent, audit, and MCP grants

Accept the agent-authority ADR. Implement grants, sensitivity, egress consent, proposal evidence validation, audit events, scoped resources/tools, revocation, idempotency, and parity contracts.

Exit: authorization and adversarial suites have zero safety failures; external clients cannot apply or broaden scope.
2.4 · feat/ai-provider-boundary

One provider, fake provider, and bounded runs

Choose the credential/egress posture, implement normalized streaming and structured output, cancellation, limits, content-free diagnostics, and deterministic fake-model fixtures.

Exit: credentials never enter project/log/export; every call discloses its exact context and destination.
2.5 · feat/candidate-fact-workflow

First useful agent workflow

Extract candidate characters, facts, and timeline events from selected scenes with evidence, then review them in the shared Inbox. This proves model → proposal → canon without editing prose.

Exit: precision meets the accepted benchmark; no candidate becomes confirmed without writer action.
2.6 · feat/continuity-review

Evidence-backed continuity

Run scoped contradiction, chronology, knowledge, naming, and world-rule passes; compare evidence; mark intentional ambiguity; generate optional fix proposals.

Exit: quality gates pass on the synthetic long-form benchmark and every citation resolves.

Phase gate: intelligence earns trust

  • Every provider call shows and enforces its context receipt, network destination, and budgets.
  • Models cannot create canonical revision events under property and adversarial tests.
  • Candidate-fact and continuity quality meets accepted precision/recall gates; unsupported claims are not presented as high-confidence fact.
  • All meaningful shipped capabilities have core use cases and UI/MCP bindings or explicit exceptions.
  • In pilot testing, at least 8/10 writers understand the review flow and prefer it to copy/paste chat for the tested tasks.

Phase gate: guidance expands the audience without slowing experts

  • Guided users reach an editable scene faster and with higher comprehension than a generic chat baseline.
  • Minimal-mode writers can start blank, hide coaching, and complete core actions keyboard-first.
  • No framework is required or used as an opaque creativity score.
  • 9/10 tested users still distinguish planned, confirmed, inferred, and proposed states.

4.3 · feat/reliability-accessibility-performance

Release-quality long projects

Harden backup/restore, diagnostics export, accessibility, import fault handling, large-project indexing, memory, bridge traffic, startup, and recovery across supported surfaces.

Exit: 120k-word benchmark meets agreed budgets; clean-install restore and fault injection pass.

Phase gate: a dependable v1

  • At least 8/10 target users complete create/import → write → proposal → export without help.
  • Pilot activation: proposed target ≥ 50% return within seven days and ≥ 30% remain active in week four.
  • All authored work is exportable without an AI provider; connection requirements and recovery state are explicit, and complete account exit is tested.
  • Accessibility, recovery, privacy, performance, and agent safety release checklists pass on each advertised platform.

Conditional Phase 5 — Advanced offline and collaboration

Basic shared projects and editorial collaboration are already in Phase 1. Do not treat these deeper capabilities as one feature.

  1. Full offline editing: complete local project replicas, queued collaboration, conflict review, encryption, and recovery.
  2. Same-scene co-authoring: add CRDT editing, multiple cursors, and convergence only if usage justifies replacing the accepted v1 scene lease.
  3. Large editorial teams: granular roles, approvals, locked scopes, workload, and organizational audit.
  4. Cross-project story universes: controlled reuse and synchronization beyond one project boundary.

Entry gate

A meaningful share of retained authors and editors repeatedly hits a limitation that the Phase 1 shared web workflow cannot serve, and the founder accepts the added security, merge, support, and cost implications in an ADR.

Industry and marketplace horizon

H.1 · research/rights-and-sharing

Prove willing supply and rights language

Test revision-pinned private submissions, editorial packages, rights metadata, creator consent, withdrawal, provenance expectations, and demand from editors or producers.

H.2 · feat/industry-interchange

Integrate before centralizing

Add deliberate exports and APIs for editors, publishers, screenwriting tools, audio, or studio workflows before building a multi-tenant marketplace.

H.3 · product/marketplace-foundation

Design a separate cloud product

Only then plan organizations, identity, catalog, moderation, search, rights, licensing, payments, tax, analytics, delivery, abuse, and dispute processes.

Proposed evidence threshold—not a forecast

  • Creation product demonstrates durable product-market fit, such as at least 30% week-eight retention among activated target writers.
  • At least 25% of active projects reach a meaningful export or edition snapshot.
  • Creators repeatedly request sharing, submission, licensing, audio, or adaptation workflows without being prompted.
  • Demand-side partners demonstrate a specific workflow and willingness to participate.
  • Rights, provenance, consent, moderation, takedown, payment, privacy, and AI-content policies are legally and operationally viable.

Permanent invariant: marketplace removal or account closure cannot strand a writer’s local project or block export. Published content is an explicit revision-pinned copy, never silent access to the private working project.

Cross-phase definition of done

  • Accepted HTML plan, current record log, focused branch, and explicit out-of-scope list.
  • Core model and use cases first; UI and MCP bindings use them rather than duplicate logic.
  • Unit, adapter contract, integration, relevant end-to-end, accessibility, privacy, and failure-state tests.
  • Real-time, reconnect, permission-revocation, version-graph, attribution, and multi-collaborator tests for every shared content slice.
  • Provider calls mocked in required CI; model-quality evals versioned and run at release gates.
  • Docs and required ADRs updated in the same branch; open decisions remain visible.
  • pnpm verify passes; manual acceptance evidence is recorded.
  • Handoff updated and plan archived only when its user outcome—not merely its code—is complete.
