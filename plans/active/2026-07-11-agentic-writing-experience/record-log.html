2026-07-11 09:20 — Trust model recommended

The planning baseline distinguishes authored manuscript, planned intent, confirmed canon, inferred analysis, and pending proposals. “Automatically kept in sync” means automatic detection with explicit writer confirmation—not silent mutation.

Built-in and external agents may read only an approved scope and create noncanonical artifacts. Canonical changes require a revision-bound proposal, visible diff, deterministic validation, explicit writer approval, an atomic apply step, provenance, and undo. This is a recommendation pending founder acceptance and should become a dedicated ADR before agent writes are implemented.

2026-07-11 11:21 — Founder set the beachhead and collaboration model

The initial beachhead is the full writing experience for novelists and aspiring novelists creating complete books, potentially across multi-book story arcs and rich shared worlds. Series scale is part of the initial domain shape, not a later specialist add-on.

The primary product is a real-time web workspace available wherever convenient. It combines a rich editor and Story Canvas with notes, edit tracking, editor collaboration, and Git-inspired safety—checkpoints, alternate versions, comparison, restoration, attribution, review, and named editions presented in writer language rather than Git mechanics.

Agents are assistants to authors and editors. They may draft ideas, edit work, and propose multiple versions of prose, but always remain in tandem with humans and cannot silently apply canonical changes.

2026-07-11 11:50 — V1 project authority and collaboration depth accepted

The founder selected online-only, server-authoritative v1 project state with minimal browser recovery rather than a complete local replica. The browser may preserve unacknowledged text through a refresh, tab crash, or brief disconnect, but normal project access and editing require a connection and Saved means durable server acknowledgement.

The founder selected live project updates, presence, comments, tracked suggestions, and version review while deferring simultaneous same-scene multi-cursor editing. V1 uses one visible active direct editor per scene plus expected-revision checks; others may read, comment, suggest, or work elsewhere concurrently.

Recorded the durable decision in ADR 0002 and updated AGENTS.md, product, architecture, operations, blueprints, roadmap, and handoff. This explicitly supersedes the repository’s earlier local-first v1 invariant and makes future backend cost/operations an open implementation decision.
