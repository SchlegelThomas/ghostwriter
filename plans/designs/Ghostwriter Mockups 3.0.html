Proposed UX 3.0 · design review

Structure · Map · Layers

Give the Canvas room
to behave like a board.

Map navigation collapses structure by default and hides Details so the board owns the screen. Tools shrink to icons with Name · shortcut tooltips. The full keyboard map stays first-class. Layers keep dive/zoom. Draft remains the only authority for reading order.

Locked for this proposal: scene creation still requires explicit book + chapter-or-Unassigned before the scene exists. Freeform covers geometry, pan, drag, links, edit, and delete after creation—not provisional Canvas-only scenes. Continues shipped UX 2.1; does not replace Mockups 2.0.

Principles

Map-first real estate without losing structure.

01Collapse structure

Wide Canvas mode shrinks the manuscript tree to glyphs; expand or peek when hierarchy work returns.

02Board is primary

Map defaults to collapsed structure + hidden Details. Chrome is thin; the board owns the viewport.

03Icon tools + tips

Compact icon toolbar and utilities; every control shows name + shortcut in a tooltip.

04Drag freely

Cards and regions move with pointer drag; Hand pans the board; arrows nudge; Fit recenters.

05Link by gesture

Drag from a connection handle to a target, or use Connect (L) for a staged keyboard path.

06Layer zoom

Double-click enters chapter/region/scene scope with camera zoom; Esc/Back and breadcrumbs exit.

07Shortcuts stay

V H S N K I R L, Space-to-pan, Escape → Select.

08Draft owns order

Canvas never silently reorders the manuscript. Drift is reviewable; structure moves stay explicit.

Navigation updates

Give the board the middle of the screen.

Map mode reclaims width from structure and Details. Mode switching, Story Trail, and layer breadcrumbs stay reachable without a permanent wide column.

2.1 / today (wide Map)

Project rail + wide manuscript tree + board + Details dock compete for width. Tool labels are text-heavy.

  • Tree often stays expanded while mapping
  • Tool dock shows name + shortcut on every button
  • Utilities are wordy chips
3.0 Map navigation

Icon project rail (44px) + structure glyph rail (48px, collapsed by default) + board + Details hidden until needed.

  • [ toggles structure; primary affordance is the »| expand control on the collapsed rail
  • Story Trail + scope crumbs live in a 40px top bar with a matching structure toggle
  • Quick Build lives on the expanded tree (or ⌘/Ctrl+Enter when structure is open)
  • Mode letters on the project rail stay icon-sized with tooltips
Structure states
  • Collapsed (Map default): ~36px; expand control »| + search + panel icon; scene glyphs optional below
  • Peek: temporary wider column on hover/focus; eases open/closed
  • Expanded: full tree + Quick Build; collapse control becomes |« or “Collapse [”
Real-estate budget (wide)
  • Project rail ~36px · Structure collapsed ~36px · Tools ~36px
  • Top bar ~40px · Utility strip ~36px · instruction line ~28px
  • Details animates to 0 width when dismissed (Map default)
  • Remaining width/height is the spatial board

Motion & feel

Quiet animation for people who live in the page.

Writers notice cadence. Transitions should feel like turning a page or sliding a desk drawer—never like a dashboard loading. Ease-out curves, short durations, and one motion at a time.

Structure open / close

Column width eases over ~380ms with cubic-bezier(.22, 1, .36, 1). Glyphs fade; tree content fades in after width starts. No bounce.

Details dock

Width + opacity ease together so the board gently grows into the freed space. Selection bar repositions with the same soft curve.

Layer zoom

~450ms camera ease into the target; siblings dim. Exit eases back. Prefer transform/opacity only for smoothness.

Tool & hover

Pressed tools and icon hover use 150–180ms color/lift. Tooltips fade in quickly; instruction strip cross-fades copy.

Cards & links

Drag uses live pointer (no lag). On drop, soft settle. New cards scale/fade in (~200ms). Link rubber-band follows the pointer; commit fades the edge in.

Menus & sheets

Context menus scale from 96% with a short fade. Narrow action sheets slide up from the bottom with the same ease-out family.

Acknowledgements

Toasts ease in from the edge and leave without stealing focus. Never animate the caret or remount Draft for shell motion.

Reduced motion

Honor prefers-reduced-motion: instant scope/chrome changes, no zoom claim, no decorative lift. Meaning stays; flourish drops.

One motion rule

Do not stack structure expand + layer zoom + dock open at once. Queue or simplify so the writer’s eye can follow a single story.

Interactive shell

Collapsed-by-default Map + icon Workbench

Hover tools for name + shortcut tooltips. Thin rails use a clear expand control. Toggle structure, Details, context menu, link-drag, and layer zoom—watch the ease. Prototype only.

Workbench chrome

Small icons, loud tooltips, quiet board.

Edge toolbar (~40px)

Icon-only. Hover/focus tooltip = Name · shortcut. Pressed tool updates the thin instruction strip and board cursor. Escape → Select.

↖ V✋ H◇ S▱ N ✦ K▧ I▢ R↗ L

Utility strip (~36px)

Lenses and viewport as icons with the same tooltip pattern. Details toggles with ] and defaults hidden while mapping.

Context menu

Right-click empty board, card, region, or link. Long-press on touch. Shortcuts listed in the menu rows. Destructive items confirm.

Empty → Create… Card → Open / Link / Archive Link → Edit / Delete

Selection bar

Compact floating bar on the selected card. Geometry stays in Details so the board stays clear.

Full user journeys

Create, link, move, edit, delete—end to end.

1. Create a note

  1. Press N or choose Note in the toolbar / empty-board context menu.
  2. Click the board (or accept default under the viewport center).
  3. Card appears selected; label focuses for rename.
  4. Acknowledgement: “Canvas note created” with Undo.

2. Create a scene (explicit placement)

  1. Press S or Create scene from the menu.
  2. Sheet asks for title and book · chapter / Unassigned (required).
  3. Optional advanced geometry stays collapsed; defaults fill x/y/size.
  4. One acknowledgement creates scene + Draft genesis + Canvas card.
  5. Writer can then drag the card freely on the board.

3. Move freely

  1. With Select, press-drag a card or region; ghost follows pointer.
  2. Release commits canvas.object.move; toast “Canvas object moved”.
  3. Hand (H) or Space-drag pans the viewport without moving objects.
  4. Arrow keys / Nudge buttons adjust by a fixed step when selected.
  5. Fit board / Fit selection recenters after large moves.

4. Drag a relationship

  1. Hover a card to reveal the out-handle (or press Link on the selection bar).
  2. Drag toward another card; rubber-band preview shows direction.
  3. Drop on a valid target → short sheet for kind / authority / optional label.
  4. Alternate path: Connect tool (L) → click source → click target → same sheet.
  5. Escape cancels with no mutation.

5. Edit & archive

  1. Double-click label or use Details / context menu Edit.
  2. Blur or Save commits metadata; prose stays in Draft.
  3. Archive from selection bar or menu → confirm → card shows archived/stale treatment.
  4. Delete link from link selection or menu → confirm → edge removed; cards remain.

6. Combo path

  1. Collapse structure → create note → drag beside scenes.
  2. Link note → scene with handle drag.
  3. Rearrange both cards; Fit board.
  4. Delete the link; keep cards.
  5. Double-click chapter aggregate (project lens) → zoom into chapter → Esc back.

Layers

Double-click zoom, not a stack of forms.

Project lens shows Chapter Aggregates. Double-click (or Enter layer) zooms the camera into that scope. Scene depth becomes Focus Stage with Draft handoffs. Reduced motion skips the zoom and changes scope instantly.

Enter

  1. Project board shows aggregate cards (derived, not new kinds).
  2. Double-click aggregate / region / scene card.
  3. Breadcrumb gains the scope; board zooms toward the target.
  4. Sibling context dims or leaves the viewport.

Exit

  1. Esc or ← Back focuses the parent control.
  2. Click a parent breadcrumb crumb.
  3. Camera eases out (or snaps if reduced motion).
  4. Selection and tool mode restore sanely (Select if Connect was mid-gesture).

Invariants

What 3.0 must not break.

Manuscript order

Tree order stays canonical. Canvas drift is visible; applying reorder needs an explicit structure command.

Explicit scene home

No scene card without book + chapter/Unassigned. Geometry is free after that.

Version domains

Canvas snapshots stay separate from Draft checkpoints. Restoring Canvas never rewrites prose.

One scene lease

Open Draft / Split / Focus Stage use the same scene ID; non-scene selection must not remount Draft.

Keyboard parity

Every toolbar and context-menu action has a keyboard path and announced name.

Thread Trace

Relationship lens remains projection—highlight and ordered text, not a second store.

Responsive & accessible

Freeform is a wide privilege.

Wide / medium

Collapse-to-rail structure, edge toolbar, pointer drag, context menus, dismissible Details. Medium may keep structure as a drawer mutually exclusive with Details.
