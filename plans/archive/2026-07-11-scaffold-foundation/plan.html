Ghostwriter delivery plan

Scaffold Ghostwriter foundation

Status: done · Branch: feat/scaffold-foundation
Delivered: PR #2 · branch preview

Intent

Create a runnable, tested TypeScript monorepo that proves the core architecture end to end: a platform-neutral writing domain, MCP access to that domain, and an Expo universal client that can deploy to Cloudflare Pages. Electron and future package boundaries are represented without pretending to ship desktop or editor functionality yet.

Acceptance criteria

  • pnpm verify typechecks, lints, and runs unit tests across the workspace.
  • packages/core has no platform imports and provides tested manuscript creation and chapter addition.
  • apps/mcp exposes a real MCP tool backed by the core package over stdio.
  • apps/client is an Expo app that exports a web build and presents the shared core’s manuscript state.
  • apps/desktop is a documented Electron shell placeholder, clearly separated from the universal client.
  • The existing GitHub production workflow can build the client; a feature-branch dev deployment is attempted through the CLI.

Tasks

  1. Set up pnpm workspaces, strict TypeScript, ESLint, Vitest, and workspace verification commands.
  2. Implement and test the platform-neutral core manuscript model.
  3. Add the MCP server with a core-backed capability tool and a smoke test.
  4. Add the Expo universal client and web-export configuration.
  5. Add explicit placeholders for desktop and future shared packages; update architecture and operations docs.
  6. Verify local commands and attempt a branch preview deployment.

Tests and verification

  • Vitest unit tests for core behavior.
  • pnpm verify and pnpm --filter client exec expo export --platform web.
  • ./scripts/deploy-dev.sh produced the branch preview linked above.
  • GitHub required check checks passed before merge.
  • A dedicated MCP process test named in the proposed plan was not added; the compile-checked status tool was accepted at merge and the gap is recorded in the log.

Documentation and ADR impact

Updated the repository, architecture, operations, planning-harness, and ADR guidance needed for the scaffold. ADR 0001 records the HTML planning workflow. The scaffold introduced no additional durable architecture decision beyond the choices already documented.

Risks and decisions

Used pnpm native workspace filtering rather than Turborepo to keep the first scaffold lean. TypeScript was pinned to stable 5.9.3 because the initially resolved 7.0.2 release was not supported by the installed TypeScript ESLint parser. Cloudflare account and token secrets were corrected outside the repository; no values were recorded.
