Ghostwriter delivery plan

HTML planning harness

Status: done · Branch: chore/html-planning-harness

Intent

Replace lightweight Markdown plans with rich HTML delivery plans that make product intent, verification, and documentation obligations explicit before implementation begins.

Acceptance criteria

  • Every future plan has visible sections for intent, todos, tasks, tests, acceptance criteria, and documentation/ADR impact.
  • Agents begin meaningful sessions by reading the HTML handoff and creating or refreshing a plan before implementation.
  • Agents can work autonomously in bounded implementation-and-verification loops while preserving user checkpoints for material decisions and external side effects.
  • The repository documents when to create or update an ADR and how that decision is linked from the plan.

Tasks

  1. Define the HTML plan, handoff, record-log, and ADR conventions.
  2. Update agent instructions with the documentation-first session and autonomous work loop.
  3. Migrate live planning entry points to HTML and preserve existing archives as historical records.
  4. Verify internal links and handoff guidance.

Tests

  • Open/parse the generated HTML documents and verify local links resolve.
  • Review the agent instructions against the delivery-plan template.

Documentation and ADRs

Create an ADR index and a decision template. This change itself is an ADR because it changes the project’s development process and agent operating model.

  • Update AGENTS.md and planning guidance

  • Migrate current handoff and README links
