Where we left off

Last updated: 2026-07-11 · Current branch: docs/agentic-writing-experience

Current state

Ghostwriter has a merged, runnable TypeScript workspace and an active product/architecture blueprint. The founder has now accepted the novel-and-series beachhead, real-time web product, Story Canvas, Git-inspired content workflow, first-class editor collaboration, and assistant-in-tandem authority model. No production behavior has changed on this branch.

Latest completed work

  • HTML planning harness established semantic delivery plans, the autonomous work loop, and ADR 0001.
  • Scaffold foundation established pnpm workspaces, strict TypeScript, core, MCP, Expo web, Electron placeholder, verification, and preview deployment through merged PR #2.

Committed foundations

  • Responsive real-time web is the primary product; Expo remains the universal client and wrappers add conveniences.
  • Tiptap (ProseMirror) editor, hosted by Expo DOM components on native.
  • GitHub Actions CI/CD; Cloudflare Pages direct upload; protected main.
  • Full novels and multi-book projects share story knowledge and cross-book arcs.
  • Story Canvas and Draft are linked views of the same scenes and story objects.
  • Authors and editors use attributable comments, suggestions, variants, compare/combine, restore, and named editions.
  • Agents propose ideas, edits, and multiple prose versions; humans decide what becomes canonical.
  • ADR 0002 sets online-only, server-authoritative v1 projects with minimal unacknowledged-work recovery and one direct scene editor at a time; full offline and same-scene multi-cursor editing are deferred.
  • Writer ownership through complete history, export, account exit, platform-neutral core, and MCP parity remains non-negotiable.

Active plan

Design the agentic writing experience now contains:

  • Experience blueprint — users, jobs, craft system, truth states, journeys, information architecture, and success measures.
  • Technical blueprint — multi-book model, server authority, browser recovery, scene leases, revisions, context, AI boundary, MCP topology, and ADR queue.
  • Agent contracts — roles, grants, proposals, provenance, safety, parity, and long-form evaluations.
  • Delivery roadmap — outcome-gated candidate plans from the writing kernel through optional industry workflows.
  • Visual mockups — founder brand direction, Story Canvas 2.0, collaborative prose variants, and responsive desktop/narrow-web concepts.

Living design source: Ghostwriter Mockups 2.0 persists outside this plan and should be read before future product, UX, editor, Canvas, imagery, linking, or workflow work.

Incomplete and awaiting founder review

The product direction is substantially accepted. These material architecture and scope decisions remain:

  1. Concrete shared backend, identity, database, real-time transport, backup, retention, deletion, and remote MCP implementation.
  2. Initial import/export formats; browser access is the accepted primary portability experience.
  3. AI provider, credentials, web egress, and local-model posture.
  4. Confirmation that marketplace, rights, collaboration, payments, and studio systems remain outside creation v1.

Recommended next step

Run the Phase 0 backend/recovery/lease architecture spike under ADR 0002, then choose concrete services and finish formats, AI posture, and marketplace boundary. Archive this blueprint only after those remaining decisions are recorded, then start the first multi-book domain plan.
