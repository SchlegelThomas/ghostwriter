Project operating system

Planning system

Plans are the shared, durable contract between the product owner and agents. They turn intent into an observable delivery agreement before implementation begins.

Layout

PathPurpose
WHERE-I-LEFT-OFF.htmlCurrent handoff; every session starts here.
template.htmlRequired structure for every new plan.
active/YYYY-MM-DD-short-slug/plan.htmlRich delivery plan for one focused branch of work.
active/.../record-log.htmlTime-stamped decisions, progress, and scope changes.
archive/Completed plan folders, preserved verbatim as project memory.

Required plan sections

Every plan.html must include: Intent, Acceptance criteria, Tasks, Tests and verification, Documentation and ADR impact, Risks and decisions, and checkable Todos. Copy the template; do not create an abbreviated substitute.

Session and delivery loop

  1. Orient: read the handoff, the active plan, its record log, and relevant product/architecture docs.
  2. Plan: for new meaningful work, create or refresh the HTML plan before touching implementation. Confirm ambiguous, consequential choices with the user.
  3. Build: work through tasks in a loop: implement a small coherent slice, run the relevant checks, inspect results, update tests and docs, then continue.
  4. Record: append real decisions, discoveries, failures, and scope changes to record-log.html. Keep plan todos truthful.
  5. Handoff: verify acceptance criteria, update the handoff, archive the completed plan, and report remaining risks.

Agents may work autonomously for extended periods inside an accepted plan. They must stop for user input when a material product, architecture, cost, security, data-loss, or external side-effect decision is not already covered by the plan.

ADRs and documentation

Plans must name every documentation impact. Create or update an ADR for durable architectural, product-platform, persistence, sync, provider, security, or operational decisions. See the ADR guide.

Historical Markdown plans

Plans archived before this system was introduced remain in Markdown as historical records. New active plans use HTML.
